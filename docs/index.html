Another Sass Interface Framework (AS-IF) is a simple, responsive CSS User Interface built with SCSS meant for bootstrapping a custom stylesheets for mobile web apps.Â The project initially started as a way to learn and practice common CSS design patterns and grew into a full framework.

View Project on Github
To start a new AS-IF project:
  1. Install Node, NPM, and Gulp in your development environment.
  2. Download or Clone the repository: git clone http://github.com/cpearson3/another-sass-interface-framework.git
  3. Install project dependencies: npm install
  4. Customize styling variables to fit your liking
  5. Run gulp to initiate the build and watch tasks for Sass and Javascript files

Project Structure

Once you download a copy of the project, you will see a directory which has the following structure:
js/
Javascript source code directory. All source is written in ES6. The main build file is scripts.js
scss/
Sass source code directory. The main build file is style.scss.
scss/_variables.scss
The variables file which defines element styles.
dist/
Location of compiled Javascript sources

Customizing the Theme Stylesheet

Grid

The grid system is uses the same markup as Materialize with a few modifications to the default configuration. The grid is based on a 12 column layout broken into rows, sections, and containers. Containers are set to 95% width by default with a max width of 1024px. Responsive column widths can be set with the .s (small), .m (medium), and .l (large) classes. For example, a column that is 75% wide on medium screens would use the .m8 class (8/12= 75%). Columns within each row are separated by a "gutter" of 1.5rem. 
$container-width: 95% !default;
$container-max-width: 1024px  !default;
$num-cols: 12 !default;
$gutter-width: 1.5rem !default;
// media queries
$small-screen-up: 601px !default;
$medium-screen-up: 993px !default;
$large-screen-up: 1201px !default;
$small-screen: 600px !default;
$medium-screen: 992px !default;
$large-screen: 1200px !default;

<div class="row">
<div class="col m8 s12">
	8 column wide area
</div>
<div class="col m4 s12">
	4 column wide area
</div>
</div>

Typography

Bootsmooth uses Modular Scale to provide scaling for font sizes. View the Github project for more information. The default font size is set to 1 rem and headings are scaled on a 1.25 ratio. Line height is set to 1.33 for headings and 1.48 for paragraphs, lists, etc. Variables can be set to change link color, hover color, text decoration, and visited color. Examples:

Buttons

Bootsmooth button styles can be applied to <button> and <a> elements with the .btn class. Modifier styles for secondary (.btn-secondary) and inverse (.btn-inverse) color styles are available. Examples: primary secondary secondary 2 complement flat inverse 
// buttons
$button-background-color: $primary-color !default;
$button-foreground-color: #FFF !default;
$button-foreground-color-secondary-1: #FFF !default;
$button-foreground-color-secondary-2: #FFF !default;
$button-foreground-color-complement: #FFF !default;
$button-hover-background-color: $primary-color-dark !default;
$button-hover-foreground-color: $button-foreground-color !default;
$button-padding: $padding $padding*2 !default;
$button-border: none !default;

<button class="btn">Primary</button>
<a class="btn-secondary">Secondary button</a>
<a class="btn-secondary-2">Secondary 2 button</a>
<a class="btn-complement">Complement button</a>
<a class="btn-inverse">Inverse button</a>

Bootsmooth offers a customizable and easy to use navigation bar component. Developers have the option of using a top or side menu that is integrated into the WordPress menu system. The mobile triggers depend on the Javascript library that is included. Any element can trigger a menu by adding the [data-nav-toggle="#MENUID"] attribute. 

// nav
$nav-background-color: $primary-color !default;
$nav-foreground-color: #FFF !default;
$nav-hover-background-color: $primary-color-dark !default;
$nav-hover-foreground-color: $nav-foreground-color !default;
$nav-title-color: $nav-foreground-color !default;
$nav-title-background-color: $nav-background-color !default;
$nav-pad: $spacing !default;
$nav-font-size: 1.0rem !default;
$nav-element-padding: 16px !default;
$nav-mobile-margin: 8px !default;
// minimum sub menu item width
$nav-sub-menu-width: 150px !default;
$nav-line-height: $line-height;


<nav class="nav">
	<div class="container">
		  <div class="nav-title" data-nav-toggle="#mainNav">NAV TITLE</div>
		<ul class="nav-menu">
			<li class="menu-item"><a href="#">Link</a></li>
			<li class="menu-item menu-item-has-children"><a href="#">Drop Dn</a>
						<ul role="menu" class="nav-menu">
					<li class="menu-item"><a href="#">Link</a></li>
					<li class="menu-item"><a href="#">Link</a></li>
				</ul>
			</li>
		</ul>
	</div>
</nav>

Padding and Spacing

Padding classes are available to provide consistent styling across components. The $padding variable sets the padding for buttons and the .padding class. The .block class is useful for providing consistent white space across a page layout. It's padding is set with the $block-padding variable. Example:
This element has padding
This element has "block padding"
$padding: 10px !default;
$block-padding: $padding * 2 !default;
		
<div class="padding">
  This element has padding
</div>
<div class="block">
  This element has "block padding"
</div>

Borders and Shadows

Borders can be added to elements with the .border class. Global settings for border radius, color, width, and style can be defined with variables. Shadows can be added with the .shadow class. Example:
Bordered and Shadow Div
// border vars and mixins
$border-color: $lightgrey !default;
$border-radius: 0px !default;
$border-width: 1px !default;
$border-style: solid !default;
$border: $border-width $border-style $border-color !default;

@mixin border($color) {
border: $border-width $border-style $color;
border-radius: $border-radius;
}

$list-item-border: $border !default;
$list-item-padding: $padding !default;
